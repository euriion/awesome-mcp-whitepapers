# Contribution Guidelines

Thank you for considering contributing to Awesome MCP Whitepapers! This document outlines the process and guidelines for contributing.

## How to Contribute

1. Fork this repository
2. Create a new branch for your contribution
3. Add your changes following the guidelines below
4. Submit a pull request

## Content Guidelines

### What to Include

This repository focuses on high-quality resources related to the Model Context Protocol (MCP):

- **Whitepapers**: Technical papers and specifications about MCP
- **Official Documentation**: Links to official MCP documentation and specs
- **Tools**: MCP servers, clients, and related tools
- **SDKs and Libraries**: Official and community SDKs for various programming languages
- **Tutorials**: Educational content helping people learn and use MCP
- **Articles**: In-depth technical articles about MCP implementations or concepts
- **Videos**: Presentations, tutorials, or talks about MCP
- **Example Projects**: Well-documented example implementations

### Quality Standards

All contributions should meet these criteria:

- **Relevant**: Directly related to the Model Context Protocol
- **High Quality**: Well-written, technically accurate, and useful
- **Maintained**: For tools and libraries, prefer actively maintained projects
- **Accessible**: Publicly available and accessible to the community
- **Well-Documented**: Resources should have clear documentation or descriptions

### What NOT to Include

- Promotional content or spam
- Unmaintained or abandoned projects (unless historically significant)
- Duplicate entries
- Resources unrelated to MCP
- Commercial products without clear technical value

## Formatting Guidelines

### List Items

Each entry should follow this format:

```markdown
- [Resource Name](URL) - Brief description of what it is/does
```

### Descriptions

- Keep descriptions concise (1-2 lines)
- Start with a verb or noun phrase
- Be objective and factual
- Avoid marketing language

### Alphabetical Order

Within each section, maintain alphabetical order by resource name.

## Suggesting New Sections

If you believe a new category would be valuable:

1. Open an issue first to discuss the proposed section
2. Explain why the section is needed
3. Provide at least 3 examples of resources that would fit the category

## Code of Conduct

- Be respectful and constructive
- Focus on the quality and relevance of contributions
- Help maintain a welcoming environment for all contributors

## Questions?

If you're unsure about whether something should be included, open an issue to discuss it!
